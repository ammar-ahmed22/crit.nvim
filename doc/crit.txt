==============================================================================
1. OVERVIEW                                                     *crit-overview*

`crit.nvim` lets you review a `crit` session from inside neovim. The agent
runs `crit session start`, gives you the id, and you drive the entire review
loop (`:CritAttach`, `:CritComment`, `:CritEdit`, `:CritDelete`, `:CritList`,
`:CritSubmit`) without ever leaving the editor.

==============================================================================
2. REQUIREMENTS                                              *crit-requirements*

  - neovim 0.10+
  - The `crit` CLI on `$PATH` (https://github.com/ammar-ahmed22/crit)
  - echasnovski/mini.diff (https://github.com/echasnovski/mini.diff)

Run `:CritDoctor` to verify both are wired up. mini.diff needs no separate
configuration; crit.nvim drives it per buffer and does not touch your own
`require('mini.diff').setup(...)`.

==============================================================================
3. SETUP                                                          *crit-setup*

Packer: >
    use {
      "ammar-ahmed22/crit.nvim",
      requires = { "echasnovski/mini.diff" },
      config = function() require("crit").setup({}) end,
    }
<

Lazy: >
    {
      "ammar-ahmed22/crit.nvim",
      dependencies = { "echasnovski/mini.diff" },
      opts = {},
    }
<

==============================================================================
4. COMMANDS                                                    *crit-commands*

                                                                 *:CritAttach*
:CritAttach {id}
    Load session {id}. `cd`s into its repo, opens the inline diff view (file
    picker + single-window unified diff) in a new tab on the session's scope
    (worktree / staged / `base...HEAD`), and paints existing comments as signs
    + virtual text. Warns if HEAD has moved since the snapshot.

                                                                 *:CritDetach*
:CritDetach
    Drop the in-memory state, clear signs, and close the diff view tab.

                                                                *:CritComment*
:[range]CritComment
    Add a comment on the current line or the supplied range. The current
    buffer must be a crit diff buffer. Comments anchor to the new side.

                                                                   *:CritEdit*
:CritEdit
    Edit the comment under the cursor. If multiple comments overlap the line,
    pick one via |vim.ui.select|.

                                                                 *:CritDelete*
:CritDelete
    Delete the comment under the cursor (with confirmation).

                                                                   *:CritList*
:CritList
    Populate the quickfix list with every draft comment. <CR> jumps to that
    comment in the inline diff view.

                                                                 *:CritSubmit*
:CritSubmit
    Open the submit window. `a`/`r`/`m` chooses the verdict; type the
    summary; `:wq` (or <C-s>) submits. Detaches on success.

                                                                   *:CritShow*
:CritShow
    Echo the attached session's id, status, scope, and instructions.

                                                                   *:CritOpen*
:CritOpen
    Open the upstream `crit open` Bubble Tea TUI in a terminal split.

                                                                 *:CritDoctor*
:CritDoctor
    Print dependency status (crit binary + version, mini.diff).

                                                                    *:CritLog*
:CritLog
    Open the crit.nvim call log buffer.

------------------------------------------------------------------------------
COMMENT EDITOR KEYS                                       *crit-comment-keys*

  The editor behaves like a normal file buffer.

    :w              stage the body (window stays open)
    :wq / :x / ZZ   save and submit, then close
    <C-s>           save and submit immediately
    <C-t>           cycle kind: comment -> question -> nit -> blocking
    :q / q / Esc    quit; submits if :w was run first, else cancels

SUBMIT WINDOW KEYS                                          *crit-submit-keys*

    a / r / m       verdict: approve / request_changes / comment
    :w              stage the summary (window stays open)
    :wq / :x / ZZ   save and submit, then close
    <C-s>           save and submit immediately
    :q / q / Esc    quit; submits if :w was run first, else cancels

==============================================================================
5. WORKFLOW                                                    *crit-workflow*

  1. Agent finishes a change set and runs `crit session start`.
  2. Agent prints the id, e.g. `crit-7f3a2c`.
  3. In nvim: >
       :CritAttach crit-7f3a2c
<     The inline diff view opens in a new tab.
  4. Pick a file in the picker (<CR>), select lines, `:'<,'>CritComment`,
     type the body, `:wq`.
  5. `:CritEdit` / `:CritDelete` as needed.
  6. `:CritSubmit`, pick verdict, `:wq`.
  7. Agent's `crit session wait` returns the review.json and proceeds.

==============================================================================
6. CONFIGURATION                                                *crit-config*

See README.md for the full default table. The most common overrides:

  bin                  path to the crit binary (default `"crit"`)
  sessions_dir         override --sessions-dir
  view.picker_width    columns for the file-picker sidebar (default 40)
  view.fold_unchanged  fold unchanged regions away from hunks (default true)
  view.context         lines kept around each hunk when folding (default 3)
  warn_on_head_drift   whether :CritAttach warns when HEAD moved (default
                       `true`)
